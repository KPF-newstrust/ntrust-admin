{{end}} {{block body()}}

일반 사용자의 요청(Request)에 따라 아래 3가지 분석결과를 리턴(Response) 해주는 API

  1. 형태소 분석 API : 사용자가 요청(Request)한 텍스트의 형태소를 분석하여 결과를 리턴(Response) 해주는 API
  2. 계량요인 추출 API : 사용자가 요청(Request)한 텍스트의 계량평가요인을 분석하여 결과를 리턴(Response) 해주는 API
  3. 트러스트 알고리즘 API : 사용자가 요청(Request)한 텍스트를 트러스트 알고리즘으로 분석하여 트러스트 점수결과를 리턴(Response) 해주는 API

트러스트 점수결과란?
언론재단 뉴스트러스트 위원회에서 정한 트러스트 알고리즘에 따라 기사를 평가한 점수결과

Open API 통합평가 이용방법

: 제목, 본문에 텍스트를 입력 후 전송한다.


RESTful API 이용방법

1. 형태소 분석 API : http://newstrust.kr/api/postagging


Field Name Type Description M/O(필수/옵션)
HEADERs
Content-Type CHARs application/json(요청이 성공인경우) O
Content-Length BODY 파트 길이 M
BODY
input CHARs 입력 문장 M
result array 형태소 분석 결과
word: 분리된 텍스트, pos: 품사 		M
unknown_words array 처리안된 문장 O
BODY 파트 예시 
{ 
  “input”: “입력 문장”,
  “result”: [{“word”:“입력”, “pos”:”명사”}, {“word”:”문장”,“pos”:”명사”}],
  “unknown_words”: []
}

2. 계량요인 추출 API : http://newstrust.kr/api/metric

Field Name Type Description M/O(필수/옵션)
HEADERs
Content-Type CHARs application/json(요청이 성공인경우) O
Content-Length BODY 파트 길이 M
BODY
sanitized_title CHARs 군더더기 제거된 제목 M
sanitized_content CHARs 군더더기 제거된 본문 M
postagging JSON array 형태소 분석 결과 { line: “분리된 문장", tags:[ {word:”단어", pos:”품사" }, {word:”단어", pos:”품사" } M
category CHARs 예상 카테고리 M
entities array 개체명 추출 결과 M
meric JSON 계량요인별 점수
바이라인:byline,
기사길이:content_length,
이미지수:image_count,
수치인용수:number_count,
제목에느낌표따옴표수:title_punc_count,
문장당평균부사수:avg_adverbs_per_sentence,
인용문수:quote_count,
문장당인용문수:quotes_per_content 		M
BODY 파트 예시 
{
  sanitized_title: “군더더기 제거된 제목",
  sanitized_content: “군더더기 제거된 본문",
  postagging: [ { line: “분리된 문장", tags:[ {word:”단어", pos:”품사" }, {word:”단어", pos:”품사" } ],
	  { line: “분리된 문장", tags:[ {word:”단어", pos:”품사" }, {word:”단어", pos:”품사" } ],
	  ….
  },
  category: “예상 카테고리",
  entities: [ “개체명1":”인명", “개체명2":”지명", ... ],
  metric: {
  content_length:1, image_count:1, number_count:1, title_punc_count:1, avg_adverbs_per_sentence:1, quote_count:1, quotes_per_content:1, ….
	  }
  } ] 
}

3. 트러스트 알고리즘 API : http://newstrust.kr/api/trust

Field Name Type Description M/O(필수/옵션)
HEADERs
Content-Type CHARs application/json(요청이 성공인경우) O
Content-Length BODY 파트 길이 M
BODY
journalism_values JSON 저널리즘 가치별 점수 : point,
계량요인별 점수 : score,
독이성:read,
투명성:transparent,
사실성:fact,
유용성:useful,
균형성:balance,
다양성:variety,
독창성:original,
중요성:important,
심층성:deep,
선정성:yellow 		M
BODY 파트 예시 
{
  journalism_values: {
	  read: { point: 5.5, scores: { image_count: 1 } },
	  transparent: { point: 5.5, scores: { byline:1, quote_count:1, title_punc_count: 1 } },
	  fact: { point: 5.5, scores: { byline:1, quote_count:1, number_count:1, image_count:1, avg_adverbs_per_sentence:1 } },
	  useful: { point: 5.5, scores: { image_count:1 } },
	  balance: { point: 5.5, scores: { quote_count:1 } },
	  variety: { point: 5.5, scores: { content_length:1, quote_count:1, image_count:1} },
	  original: { point: 5.5, scores: { title_punc_count:1, image_count:1 } },
	  important: { point: 5.5, scores: { content_length:1, quote_count:1, number_count:1 } },
	  deep: { point: 5.5, scores: { content_length:1, number_count:1, image_count:1 } },
	  yellow: { point: 5.5, scores: { byline:1, title_punc_count:1, title_adverb_count:1, avg_adverbs_per_sentence:1, quotes_per_content:1 }
	  }
  }
}
